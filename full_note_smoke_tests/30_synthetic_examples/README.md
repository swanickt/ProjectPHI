# ProjectPHI Synthetic Smoke Tests

## Scope

This smoke test uses a batch of 30 synthetic notes, across 14 fictional patient ids:

- 27 expected successful rows (i.e., de-identified notes);
- 3 intentional failure rows with missing alias metadata;
- repeated patient IDs to check stable date and stable patient-name behavior
  across multiple notes;
- breast oncology, imaging, radiation, pathology, copied correspondence,
  administrative, role-context, and title-context false-positive scenarios.

MTSamples pages were used as format/category references for AI-written synthetic examples.

## How The Notes Were Created

The notes in `input_notes.csv` are synthetic AI-generated (gpt-5.5) smoke-test notes. They are not copied patient notes, not copied
clinical examples, and not real PHI. Some note wording is intentionally artificial because gpt-5.5 was asked to include ProjectPHI edge cases, such as title-context false positives, protected
  clinical terms, stable alias behavior, synthetic identifiers, and failure-path rows. These notes are therefore smoke-test stress cases, not examples of natural
  clinical documentation style.


The note set was created by:

1. Having gpt-5.5 review public MTSamples pages listed in `source_manifest.csv` to identify
   broad document styles and clinical-note categories, such as breast oncology
   consults, SOAP/progress follow-up notes, radiation oncology follow-up notes,
   hereditary-risk/discharge-style notes, and hematology/oncology
   procedure/pathology-style categories.
2. Then, having gpt-5.5 write new synthetic notes locally in those broad styles, using invented
   patient IDs, names, emails, phone numbers, identifiers, dates, facilities,
   clinician names, family names, and local-code-like strings.
3. gpt-5.5 was prompted to add targeted stress cases based on ProjectPHI's expected behavior:
   stable dates across repeated notes for the same patient, explicit patient
   aliases, missing-alias failure rows, copied-correspondence names,
   clinician/family names, title-context false positives, protected clinical
   terms, synthetic custom-regex identifiers, contact details, and audit
   privacy checks.

The MTSamples references were used only to help choose headings, note genres, and
clinical documentation patterns to imitate. Their note bodies were not copied,
paraphrased as patient examples, or stored anywhere in the repo. The clinical content
here is intentionally simplified and synthetic, with obvious placeholder-style
identifiers such as `SYN-EVAL-*`, test emails, and invented names.

This folder is therefore useful as a local smoke review of ProjectPHI behavior,
but it is not a gold-standard evaluation set and should not be treated as a
clinical corpus.

## Files

- `source_manifest.csv`: source/category references used to choose smoke-test
  styles.
- `patient_aliases.csv`: synthetic alias manifest.
- `provider_aliases.csv`: synthetic provider alias manifest using actual names
  or name components only, such as `Chen` or `Lena Shore`.
- `custom_doctor_names.json`: synthetic provider-name components passed to
  pyDeid custom doctor-name hooks by the local runner.
- `custom_regexes.json`: synthetic custom regex configuration.
- `input_notes.csv`: 30 synthetic notes.
- `deidentified_output.csv`: pipeline output generated by the local runner.
- `audit_output.csv`: internal audit output generated by the local runner.
- `run_smoke_test.py`: folder-local runner that calls `deidentify_csv(...)`
  with stable dates, stable patient-name aliases, custom regexes, and custom
  doctor-name lists.
- `readable_review/`: human-readable copies of the current input notes,
  de-identified output notes, failed rows, and audit rows. These files are
  generated from the CSVs for manual review only; the note text and audit field
  values are copied exactly from the CSVs and separated by patient/note for
  readability.


## Pipeline Command

```sh
time PROJECT_PHI_DATE_SHIFT_SECRET=local-eval-date-secret \
PROJECT_PHI_PATIENT_NAME_SECRET=local-eval-name-secret \
PROJECT_PHI_PROVIDER_NAME_SECRET=local-eval-provider-secret \
uv run python full_note_smoke_tests/30_synthetic_examples/run_smoke_test.py
```

The secrets above are synthetic local-evaluation values only.
